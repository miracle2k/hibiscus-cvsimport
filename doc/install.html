Installation

Systemvoraussetzungen

  • Java 1.4 oder hoeher (JRE oder SDK)
    Geben Sie "java -version" an einer Shell (Windows: Eingabeaufforderung) ein um die Version zu ueberprüfen.
    Java können Sie von http://java.sun.com/j2se/index.jsp herunterladen.
  • Jameica
    Kann von http://www.willuhn.de/projects/jameica/ heruntergeladen werden.

Installation und Konfiguration

  1. Entpacken Sie die ZIP-Datei jameica-<version>.zip in das von Ihnen gewünschte Programm-Verzeichnis, z.Bsp.:
    Windows C:\Progamme\
    Linux/Linux-AMD64/MacOS /opt
    Anschliessend sollte dort ein Verzeichnis "jameica" existieren.
  2. Entpacken Sie ZIP-Datei hibiscus-<version>.zip in das Verzeichnis "plugins" innerhalb Ihrer Jameica-Installation.
    z.Bsp.:
    Windows C:\Progamme\jameica\plugins
    Linux/Linux-AMD64/MacOS /opt/jameica/plugins
    Hierbei wird ein Unterverzeichnis "hibiscus" angelegt.
  3. Starten Sie nun Jameica mit dem entsprechenden Startscript für Ihre Betriebssystem:
    Windows C:\> cd \Progamme\jameica
    C:\Programe\jameica> jameica.bat
    Linux $> cd /opt/jameica
    $> ./jameica.sh
    Linux-AMD64 $> cd /opt/jameica
    $> ./jameica-amd64.sh
    MacOS $> cd /opt/jameica
    $> ./jameica-macos.sh
    Achten Sie bitte darauf, dass Sie sich beim Aufruf des Startscripts im Programmverzeichnis befinden müssen. Andernfalls wird Jameica wichtige Programmbibliotheken nicht finden können und den Start mit einem Fehler abbrechen. Sollten Sie eine Verknüpfung in Ihrem Startmenü oder auf dem Desktop anlegen wollen, so beachten Sie hier bitte, das Arbeitsverzeichnis ebenso anzupassen.

    Wenn Sie die Anwendung nun zum ersten Mal starten, werden Sie zur Neueingabe eines Master-Passwortes aufgefordert. Mithilfe dieses Passwortes werden alle streng vertraulichen Informationen verschlüsselt. Wählen Sie also bitte ein hinreichend sicheres Passwort. Merken Sie es sich gut. Sie können es nachträglich auch über Datei»Master-Passwort ändern ändern.
  4. Der Start sollte nun einige Sekunden länger dauern. Währenddessen wird im Splashscreen der aktuelle Installations-Status des Plugins angezeigt.
  5. Ihnen wird nun ein Dialog mit den ersten Schritten angezeigt. Klicken Sie gemäss den Anweisungen auf Sicherheitsmedium einrichten, um Ihren Chipkartenleser, Ihre Schlüsseldiskette oder PIN/TAN-Konfiguration einzurichten. Klicken Sie dort anschliessend auf den Test-Button, um die korrekte Einrichtung zu prüfen.

    Nur Linux/Linux-AMD64: Beachten Sie bitte bei der Verwendung eines Chipkartenlesers, dass Sie mit Ihrem Benutzer-Account Schreibrechte auf dem entsprechenden Device (serieller Port) benötigen. Ändern Sie diese ggf. zum Beispiel durch folgend Befehl (als Root) ab: 
    $ chgrp users /dev/ttyS0 /dev/ttyS1; chmod 775 /dev/ttyS0 /dev/ttyS1
  6. Wechseln Sie in der Navigation zu Jameica»Hibiscus»Konten um die Liste der verfügbaren Konten anzuzeigen. Wählen Sie in dem Kasten "Konten automatisch aus Sicherheitsmedium ermitteln" das soeben konfigurierte Medium aus und klicken Sie auf "Daten aus Medium lesen". Insofern Ihre Bank dies unterstützt werden nun die verfügbaren Bankverbindungen automatisch ermittelt. Sie werden ggf. zur Eingabe Ihrer PIN aufgefordert. Falls Ihre Bank das Abrufen der Kontenliste nicht unterstützt, legen Sie die gewünschten Bankverbindungen bitte manuell über den Button "Neue Bankverbindung" an.
  7. Nachdem Sie durch Doppelklick auf eines der Konten die Detail-Ansicht geöffnet haben, können Sie über "Saldo und Umsätze abrufen" wählen.

Sicherheitshinweise

  1. Erstellen Sie regelmässig Backups Ihres Benutzer-Verzeichnisses.
    Windows C:\Dokumente und Einstellungen\<benutzername>\.jameica
    Linux/Linux-AMD64/MacOS /home/<benutzername>/.jameica
  2. Hibiscus verwendet eine integrierte SQL-Datenbank (McKoi) zur Speicherung der Konten, Bankverbindungen, Salden und Ueberweisungen. Die Daten werden im "Benutzer"-Verzeichnis von Jameica abgelegt. Das ist i.d.R.
    Windows C:\Dokumente und Einstellungen\<benutzername>\.jameica\hibiscus\db
    Linux/Linux-AMD64/MacOS /home/<benutzername>/.jameica/hibiscus/db
    Diese Informationen werden unverschlüsselt in der Datenbank abgelegt. Passen Sie also ggf. die Dateiberechtigungen so an, dass Sie nur von Ihrem Benutzer-Account gelesen werden können bzw. achten Sie darauf, dass die Daten nicht fuer Dritte zugaenglich sind.
  3. String vertrauliche Informationen wie Passwörter, PIN oder TANs werden jedoch grundätzlich gar nicht oder verschlüsselt gespeichert.

Hardware Support-Matrix

Durch die verschiedenen auf dem Markt erhältlichen Chipkartenleser und unterschiedlichen Betriebssysteme ergibt sich eine Vielzahl von Kombinationsmöglichkeiten. Derzeit ist hiervon nur ein Bruchteil getestet. Prinziell sollte die Software aber überall funktionieren. Mangels Testhardware und Software ist es mir leider nicht möglich, all diese Kombinationen zu prüfen.

Unter MacOS werden derzeit leider keine Chipkartenleser sondern nur Schlüsseldiskette und PIN/TAN unterstützt.
