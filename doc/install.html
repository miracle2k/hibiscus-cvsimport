Installation

Systemvoraussetzungen

  • Java 1.4 oder hoeher (JRE oder SDK)
    Geben Sie "java -version" an einer Shell (Windows: Eingabeaufforderung) ein um die Version zu ueberprüfen.
    Java können Sie von http://java.sun.com/j2se/index.jsp herunterladen.
  • Jameica
    Kann von http://www.jameica.org heruntergeladen werden.

Installation und Konfiguration

  1. Kopieren Sie den Inhalt dieser ZIP-Datei in das Verzeichnis "plugins" innerhalb Ihrer Jameica-Installation.
    z.Bsp.:
    Windows C:\Progamme\jameica\plugins
    Linux /opt/jameica/plugins
  2. Starten Sie nun Jameica. Falls Sie die Anwendung zum ersten Mal starten, werden Sie zur Neueingabe eines Master-Passwortes aufgefordert. Mithilfe dieses Passwortes werden alle streng vertraulichen Informationen verschlüsselt. Wählen Sie also bitte ein hinreichend sicheres Passwort. Merken Sie es sich gut. Sie können es nachträglich auch über Datei»Master-Passwort ändern ändern.
  3. Der Start sollte nun einige Sekunden länger dauern. Währenddessen wird im Splashscreen der aktuelle Installations-Status des Plugins angezeigt.
  4. Wählen Sie nun im Menu Plugins»Hibiscus»Einstellungen. Klicken Sie in der Liste der verfügbaren Sicherheitsmedien (Chipkarte, PIN/TAN, Schlüsseldiskette) doppelt auf das gewünschte Medium um den Konfigurations-Dialog zu öffnen und legen Sie ein neues Sicherheitsmedium an. Klicken Sie anschliessend auf den Test-Button um die korrekte Einrichtung zu prüfen.
    Nur Linux: Beachten Sie bitte bei der Verwendung eines Chipkartenlesers, dass Sie mit Ihrem Benutzer-Account Schreibrechte auf dem entsprechenden Device (serieller Port) benötigen. Ändern Sie diese ggf. zum Beispiel durch folgend Befehl (als Root) ab: 
    $ chgrp users /dev/ttyS0 /dev/ttyS1; chmod 775 /dev/ttyS0 /dev/ttyS1
  5. Wechseln Sie in der Navigation zu Jameica»Hibiscus»Konten um die Liste der verfügbaren Konten anzuzeigen. Wählen Sie in dem Kasten "Konten aus Medium lesen" das soeben konfigurierte Medium aus und klicken Sie auf "Daten aus Medium lesen". Insofern Ihre Bank dies unterstützt werden nun die verfügbaren Bankverbindungen automatisch ermittelt. Sie werden ggf. zur Eingabe Ihrer PIN aufgefordert. Falls Ihre Bank das Abrufen der Kontenliste nicht unterstützt, legen Sie die gewünschten Bankverbindungen bitte manuell über den Button "Neue Bankverbindung" an.
  6. Nachdem Sie durch Doppelklick auf eines der Konten die Detail-Ansicht geöffnet haben, können Sie über "Saldo und Umsätze abrufen" wählen.

Installation zusätzlicher Sicherheitsmedien

Hibiscus enthält bereits die Unterstützung für das Sicherheitsmedium DDV (Chipkarte mit synchronen Schluesseln), Schlüsseldiskette und PIN/TAN. Die Installation weiterer Sicherheitsmedien (insofern diese als Plugin für Hibiscus existieren) beschränkt sich i.d.R. auf das Kopieren der Dateien in das Verzeichnis "lib". z.Bsp.:
Windows C:\Progamme\jameica\plugins\hibiscus\lib
Linux /opt/jameica/plugins/hibiscus/lib
Beim erneuten Start von Hibiscus werden diese automatisch erkannt unter Plugins»Hibiscus»Einstellungen aufgelistet. Sollte das installierte Medium dort nicht erscheinen, prüfen Sie die Log-Ausgaben von Jameica (entweder durch Öffnen der Log-Datei im Work-Verzeichnis oder durch Klick auf den linken Bereich der Statusleiste von Jameica). Senden Sie diese bitte an den Autor.
Weitere Hinweise zur Installation entnehmen Sie bitte der Dokumentation des jeweiligen Sicherheitsmediums.

Sicherheitshinweise

Hibiscus verwendet eine integrierte SQL-Datenbank (McKoi) zur Speicherung der Konten, Bankverbindungen, Salden und Ueberweisungen. Die Daten werden im "Work"-Verzeichnis von Jameica abgelegt. Das ist i.d.R.
Windows C:\Dokumente und Einstellungen\<benutzername>\.jameica\hibiscus\db
Linux /home/<benutzername>/.jameica/hibiscus/db
Diese Informationen werden unverschlüsselt in der Datenbank abgelegt. Passen Sie also ggf. die Dateiberechtigungen so an, dass Sie nur von Ihrem Benutzer-Account gelesen werden koennen bzw. achten Sie darauf, dass die Daten nicht fuer Dritte zugaenglich sind. String vertrauliche Informationen wie Passwörter, PIN oder TANs werden jedoch grundätzlich gar nicht oder verschlüsselt gespeichert.

Hardware Support-Matrix

Durch die verschiedenen auf dem Markt erhältlichen Chipkartenleser und unterschiedlichen Betriebssysteme ergibt sich eine Vielzahl von Kombinationsmöglichkeiten. Derzeit ist hiervon nur ein Bruchteil getestet. Prinziell sollte die Software aber überall funktionieren. Mangels Testhardware und Software ist es mir leider nicht möglich, all diese Kombinationen zu prüfen.

Bisher getestet:
Chipkartenleser Betriebssystem Status
Towitoko Kartenzwerg / Chipdrive Micro Linux (GLIBC 2.2) OK
Towitoko Kartenzwerg / Chipdrive Micro Linux (GLIBC 2.3) OK
Towitoko Kartenzwerg / Chipdrive Micro Windows CTAPI-Treiber ungetestet
Kaan Standard Plus USB Linux (GLIBC 2.3) OK
Kaan Standard Plus USB Windows OK
Chipdrive Pinpad / SCM SPR 332 Windows OK
Chipdrive Pinpad / SCM SPR 332 Linux kein CTAPI-Treiber gefunden
Sollten Sie Hibiscus erfolgreich mit einem hier nicht aufgeführten Chipkartenleser im Einsatz haben, dann schreiben Sie mir das bitte an hibiscus@willuhn.de damit ich diese Liste verfollständigen kann.
